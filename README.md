# Классическая консольная мини-игра "Остров Проклятых"

## Введение

В процессе разработки проекта я взяла за основу классический пример игры от Светланы Сергеевны.

Бой с финальным боссом мог быть взят из второй лабораторной работы, однако, я сочла не очень интересным большое количество
действий, выполняемых героями рандомно, и переписала механику боя.

---

# 1. Лор и концепция игры

Игрок попадает на одинокий остров в бескрайнем океане. Чтобы выбраться отсюда, ему придется пройти испытания в нескольких
локациях и сразиться с финальным боссом)(испытания несложные, все-таки это лишь небольшая консольная игра, а не полноценная крутая сюжетная штука). 

Испытания:

- отгадать загадки,
- составить слово из букв,
- отгадать число.

---

## Концовка 1 (худшая) — Игрок теряет все свои артефакты в процессе игры и проклятие поглощает его.

Такая концовка зависит от количества артефактов, которые игрок собрал в процессе игры, а также от успешности его прохождения
испытаний.

---

## Концовка 2 (плохая) — Игрок проигрывает Боссу в финальной битве, проклятие снять не удается.

Такая концовка зависит лишь от исхода финальной битвы, на нее не влияет количество артефактов.

---

## Концовка 3 (хорошая) — Игрок побеждает Босса, но с небольшим кол-вом артефактов, остров медленно восстанавливается от проклятия.

На эту концовку влияет количество собранных во время игры (или сохраненных с прошлой игры) артефактов, а также исход
финальной битвы. Такая концовка выпадет при победе с 1-2 артефактами.

---

## Концовка 4 (хорошая) — Игрок побеждает Босса и уплывает с острова.
На эту концовку также влияет количество собранных во время игры (или сохраненных с прошлой игры) артефактов, а также исход
финальной битвы. Такая концовка выпадет при победе с 3-4 артефактами.

---

## Концовка 5 (лучшая) — Игрок побеждает Босса и становится новым Хранителем Острова.
На эту концовку все еще (не опять, а снова) влияет количество собранных во время игры (или сохраненных с прошлой игры) артефактов, а также исход
финальной битвы. Такая концовка выпадет при победе с 5+ артефактами.
---

# 2. Игровые артефакты

Артефакты в игре делятся на два типа (и записываются в два разных файла):

1. Стартовые артефакты (один из них выпадает рандомно в самом начале игры)
2. Артефакты, приобретаемые после испытаний.


---

# 3. Инструкция по использованию

---

## Запуск игры

Требуется Python **3.10+**

```bash
python main.py
```

---

## Работа с пользователями

Сохраняется:
- прогресс игрока
- полученные артефакты

---
## Геймплейный цикл

1. Начало игры
2. Выбор локации 
3. Прохождение испытания
4. Получение/потеря артефакта (если потерян последний артефакт - худшая концовка)
5. Возможность вернуться в главное меню/ возможность начать финальную битву (предлагается от 2+ артефактов)
6. Финальная битва
7. Проверка победы/поражения игрока
8. Концовка

**Игра завершается при:**

- достижении одной из пяти концовок,
- принудительном выходе игрока.

# 4. Техническое описание проекта
Структура проекта
```
project/
 ├─ main.py                — главный сценарий
 ├─ player.py              — класс игрока
 ├─ boss.py                — класс босса
 ├─ start_items.txt        — файл стартовых артефактов (создается автоматически)
 ├─ spirit_items.txt       — файл приобретаемых артефактов (создается автоматически)
 └─ island_log.txt         — сохраненный прогресс игры (создается после окончания)
```


---
## Технические особенности реализации

* save_game() - сохраняет прогресс в JSON-файл
* discard_progress() - возвращает все артефакты в копилку (спрашивает подтверждение)
* в каждой локации появляется меню сохранения
* у Игрока в финальной битве есть специальная атака 
* двухфазовая система боя у Босса, во второй фазе - усиление
* визуальные индикаторы здоровья- сердечки
* полностью консольный интерфейс
